# pdf2md 召回率评估

**目的**：改 pdf2md 后重跑，对比 README 里记录的召回比基线，看有没有退步/提升。准确率依然人工抽查（和 README 记录的 37 数字抽查一致的方法）。

## 基线（README 里记录）

| PDF | 召回比 | 备注 |
|---|---|---|
| 天津真格（45页，中文）| 113% | vision 补充了 native 遗漏 |
| 苏州济峰（21页，中文）| 104% | — |
| IDG（38页，英文）| 95% | 少量页眉页脚未提取 |
| 昆山峰瑞（11页，字体乱码）| 134% | vision 还原了乱码数字 |
| 广州越秀（32页，扫描件）| — | OCR 生成，无 PyMuPDF 基线 |

基线定义：`len(strip_markdown(MD)) / len(PyMuPDF 抽出的 PDF 原文)`，**单位是字符数**。

> 召回比 > 100% 是正常的——vision 能补上 PyMuPDF 漏抽的内容（图片里的文字、乱码数字等）。

## 工作流

```bash
# 1. 改 pdf2md 代码
# 2. 重跑 pdf2md 产出新 MD（可能需要 --force 覆盖）
python pdf2md.py ~/Desktop/归档/ -o ~/Desktop/归档/output/ --force --stage extract

# 3. 跑召回率对比
python -m eval.run_eval
```

输出：

```
PDF               现在    README 基线             Δ     base len      md len
────────────────────────────────────────────────────────────────────────────────
天津真格         112.7%            113%         -0.3pp =       72654       81914
苏州济峰         104.5%            104%         +0.5pp =       45123       47153
IDG               95.2%             95%         +0.2pp =       91237       86858
昆山峰瑞         134.3%            134%         +0.3pp =        8321       11176
广州越秀            —                —          (无基线)          0       23842

提示：准确率（数字/字段正确性）仍需人工抽查——脚本只衡量召回。
```

**判读**：
- `Δ ≥ 0` + 人眼抽查通过 → 这次 pdf2md 改动可以提交
- `Δ < 0`（回退）→ 打开对应 PDF 的 MD 人眼看哪里漏了

## 局限

- **只衡量字符数量**，不看内容正确性。MD 可能出现"长度一致但金额写错"，召回率看不到（靠你人眼抽查兜底）。
- **基线不完美**：PyMuPDF 对字体乱码 PDF 抽的是乱码字符，对扫描件抽不到——所以 README 基线里昆山峰瑞 134%、广州越秀 N/A，这些是已知的，继续保持口径一致即可。
- **没有自动 Δ 判断是否 commit 的逻辑**。决策权完全在你手上：看数字 + 人眼验收 + commit。

## 文件

```
eval/
├── baseline.py      # PyMuPDF 抽基线文本
├── md_parser.py     # 剥 Markdown 标记
├── normalize.py     # 文本归一化（NFKC/去千分位/压空白/小写）
├── run_eval.py      # 入口：跑 5 份 PDF 的 recall + 对比 README
├── config.yaml      # 本机绝对路径（不入库）
├── config.yaml.example
└── tests/           # pytest 基础测试
```
